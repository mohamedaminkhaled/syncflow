# Contributing to SyncFlow

Thank you for considering a contribution. This project is being built in public; all discussion happens via GitHub Issues and Pull Requests.

## Getting started

1. Open or find an issue describing the bug or feature.
2. Comment on the issue so work is not duplicated.
3. Fork the repo, create a branch from `main`, and open a PR when ready.

## Pull request rules

- One logical change per PR.
- All commits must be signed (`git commit -s`).
- PRs require green CI and at least one approving review before merge.
- Update `README.md` if your change alters local development steps.

## Code style

- Python: PEP 8, `ruff`/`black` enforced in CI.
- Commits: conventional commits (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`).

## Questions?

Open a Discussion or Issue. Responses may be slow during early weeks.
